# ⏱ Чекпоинт-Таймер для BeamNG.drive

Мод добавляет в игру **App UI** — небольшую панель на экране, с помощью которой можно:

1. **Записывать маршруты** — отметить точку старта, любое количество промежуточных чекпоинтов и точку финиша. Координаты сохраняются в конфиг-файл для текущей карты.
2. **Гонять по маршруту** — таймер автоматически засекает время от старта, фиксирует дельту (разницу) между каждой парой точек и считает общее время заезда.
3. **Видеть ворота в 3D-мире** — старт (зелёные ворота), чекпоинты (жёлтые), финиш (красные) рисуются прямо в мире как прямоугольные рамки ~2.5м шириной.
4. **Вести историю** — все заезды сохраняются, их можно развернуть и посмотреть сплиты по каждой точке.

---

## 📦 Установка

### Способ 1 — распакованный (для разработки, быстрее обновлять)

1. Скопируйте папку мода (внутри которой лежат `lua/`, `ui/`, `scripts/`, `mod_info/`, `settings/`) в:

   **Windows:**
   ```
   C:\Users\<ИМЯ_ПОЛЬЗОВАТЕЛЯ>\AppData\Local\BeamNG.drive\<версия>\mods\unpacked\checkpointTimer\
   ```
   **Linux:**
   ```
   ~/.local/share/BeamNG.drive/<версия>/mods/unpacked/checkpointTimer/
   ```

   > 💡 Чтобы найти папку `<userfolder>` в игре: **меню → Options → About → Open user folder**.

2. Запустите игру.
3. В игре зайдите в **Vehicle Setup** (иконка гаечного ключа) → вкладка **Apps** → найдите в списке **«Checkpoint Timer»** → перетащите на экран.
4. Перезагрузите Lua-скрипты: **Ctrl+L** (GELUA) и **F5** (Main UI).

### Способ 2 — запакованный (для распространения)

1. Запакуйте **содержимое** папки мода в ZIP. Структура внутри ZIP должна быть такой:
   ```
   CheckpointTimer.zip
   ├── lua/
   ├── ui/
   ├── scripts/
   ├── mod_info/
   └── settings/
   ```
   *(то есть папки `lua/`, `ui/` и т.д. лежат в корне архива, а не внутри ещё одной папки)*
2. Положите ZIP-файл в `<userfolder>/mods/`.
3. Запустите игру, добавьте App через **Vehicle Setup → Apps**.

## 🎮 Как пользоваться (пошагово)

### Шаг 1. Добавить панель на экран

В игре: **иконка гаечного ключа** (справа от выбранной машины) → вкладка **Apps** → найдите **Checkpoint Timer** → кликните иконку или перетащите её в нужное место экрана.

На экране появится тёмная панель с заголовком **«⏱ ЧЕКПОЙНТ-ТАЙМЕР»**.

### Шаг 2. Записать маршрут

Маршрут = **старт + N чекпоинтов + финиш**. Минимум — старт и финиш (без чекпоинтов).

1. Сядьте в машину, подъедьте к точке, где должен быть **старт**.
2. В поле «Имя маршрута (необязательно)» впишите название, например `Highway_Sprint`. Если не ввести — будет сгенерировано автоматически (`Route_HHMMSS`).
3. Нажмите зелёную кнопку **«📍 Новый»**.
   - Появится красная надпись «🔴 ЗАПИСЬ: Highway_Sprint» и подпись «Ожидание записи СТАРТА».
   - На самом деле старт уже записан той точкой, где стояла машина при нажатии. Подпись означает «теперь подъезжайте к чекпоинту».
4. Подъедьте к первой промежуточной точке (чекпоинту). Нажмите **«▶ Точка»**.
   - В логе появится чекпоинт №1, а в 3D-мире — жёлтые ворота «ЧК1».
5. Подъедьте к следующему чекпоинту → снова **«▶ Точка»**. И так для всех промежуточных точек.
6. Подъедьте к точке финиша → нажмите **«🏁 Финиш»**.
   - Маршрут сохранится в файл `settings/checkpointTimer/<имя_карты>.json`.
   - В списке «Маршруты на карте» он появится сразу.

**Отмена записи:** если передумали — нажмите **«✖ Отмена»**, ничего не сохранится.

### Шаг 3. Проехать заезд

1. В списке «Маршруты на карте» кликните по нужному маршруту — он подсветится жёлтым.
2. В мире появятся **3D-ворота** (вертикальные прямоугольные рамки ~2.5м шириной и 4м высотой):
   - 🟢 **зелёные ворота** с надписью «СТАРТ» — точка старта
   - 🟡 **жёлтые ворота** с надписями «ЧК1», «ЧК2», … — промежуточные чекпоинты
   - 🔴 **красные ворота** с надписью «ФИНИШ» — точка финиша
   - 🟦 Синяя пунктирная линия на земле соединяет все ворота — это линия маршрута.
   - Ворота автоматически повёрнуты перпендикулярно направлению движения (по последовательности точек).
3. Есть два способа проехать:

   **Авто-режим (рекомендуется):**
   - Просто въезжайте в зелёные ворота старта — таймер запустится сам.
   - Далее проезжайте по очереди через жёлтые ворота чекпоинтов — каждый фиксируется автоматически.
   - В конце въезжайте в красные ворота финиша — заезд завершится.
   - Подъезжать к чекпоинтам нужно **по порядку** (ЧК1 → ЧК2 → … → ФИНИШ).
   - Триггер срабатывает, когда машина оказывается в радиусе (по умолчанию 3м) от центра ворот — не обязательно ехать точно по центру.

   **Ручной режим:**
   - Нажмите **«▶ Старт»** в начале заезда.
   - На каждом чекпоинте нажимайте **«◆ Чекп.»**.
   - На финише — **«🏁 Финиш»**.
   - Этот режим удобен, если авто-детект не срабатывает (например, чекпоинты слишком близко).

4. Во время заезда сверху панели показывается **крупный таймер** и счётчик «Чекпоинтов: X / Y».
5. После финиша в разделе «Сплиты» появятся строки:
   - **Точка** — СТАРТ / ЧК1 / ЧК2 / … / ФИНИШ
   - **Дельта** — время от предыдущей точки до этой
   - **Итого** — накопительное время от старта
   - **Разница** — на сколько быстрее/медленнее, чем ваш лучший сплит на этой точке:
     - 🟢 зелёным — если побили свой рекорд на этом отрезке
     - 🔴 красным — если проехали хуже, чем в лучший раз
     - `—` — если это первый заезд (рекорда ещё нет)

### Шаг 4. Посмотреть историю

В разделе «📜 История заездов» — последние 20 заездов. Клик по строке разворачивает все сплиты этого заезда. Кнопка **«🗑 Очистить историю»** внизу — стирает все записи (маршруты при этом остаются).

## ⚙ Настройки

В нижней части панели:

| Поле / кнопка | Что делает |
|---|---|
| **Радиус (м)** | Радиус сферической триггер-зоны в метрах. По умолчанию 3 м. Больше — легче зацепить зону, но можно случайно «пройти» чекпоинт, проехав мимо. Меньше — точнее, но нужно подъезжать вплотную. |
| **Частота (Гц)** | Как часто Lua проверяет позицию машины (20 раз в секунду по умолчанию). Выше — точнее фиксация момента прохождения, но больше нагрузка. |
| **Ворота ширина** | Ширина 3D-ворот в метрах. По умолчанию 2.5 м. Это чисто визуальный параметр — на триггер не влияет (за него отвечает «Радиус»). |
| **Ворота высота** | Высота 3D-ворот в метрах. По умолчанию 4 м. |
| **👁 Зоны: ВКЛ/ВЫКЛ** | Показывать / скрыть 3D-ворота в мире. |
| **🐞 Дебаг: ВКЛ/ВЫКЛ** | Включить дополнительный лог в консоль каждый кадр. |
| **🔄 Перезагрузить** | Заново прочитать конфиг-файлы с диска (если вы вручную правили JSON). |
| **🔧 Диагностика 3D** | Выводит в лог информацию о доступности `debugDrawer`, `vec3`, `ColorF` — используйте, если 3D-ворота не отображаются. |

## 🖥 Нижняя панель (быстрые команды в лог)

| Кнопка | Что делает |
|---|---|
| **Список** | Выводит в лог все маршруты текущей карты |
| **Инфо** | Выводит в лог текущий статус (карта, маршрут, идёт ли заезд) |
| **Лог** | Выводит в лог всю историю |
| **Справка** | Выводит в лог список всех консольных команд |
| **Синхр** | Принудительно запросить у Lua свежий статус (если UI завис) |

> **Лог** — это консоль разработчика. Открывается клавишей **`~`** (тильда, под Esc) или через **F11 → Console**.

## 💻 Консольные команды

Все команды вызываются из консоли так:

```lua
checkpointTimer.ct_command("help")
```

Полный список:

| Команда | Что делает |
|---|---|
| `help` | Справка по командам |
| `rec start <имя>` | Начать запись нового маршрута с указанным именем |
| `rec next` | Записать текущую позицию как следующую точку |
| `rec finish` | Записать финиш и сохранить маршрут |
| `rec cancel` | Отменить запись |
| `sel <имя>` | Выбрать маршрут |
| `del <имя>` | Удалить маршрут |
| `list` | Список маршрутов текущей карты |
| `info` | Текущий статус |
| `start` | Ручной старт заезда |
| `cp` | Ручной проход чекпоинта |
| `finish` | Ручной финиш |
| `reset` | Сбросить текущий заезд |
| `history` | История в лог |
| `clearhistory` | Очистить историю |
| `debug` | Переключить дебаг-режим |
| `zones` | Переключить 3D-визуализацию ворот |
| `radius <метры>` | Установить радиус триггер-зоны (напр. `radius 3`) |
| `hz <герцы>` | Установить частоту проверки (напр. `hz 30`) |
| `gatewidth <м>` | Установить ширину ворот (напр. `gatewidth 3.5`) |
| `gateheight <м>` | Установить высоту ворот (напр. `gateheight 5`) |
| `diag` | Диагностика — проверяет доступность debugDrawer, vec3, ColorF |
| `push` | Принудительно отправить статус в UI |
| `reload` | Перезагрузить конфиг текущей карты |

## ❓ Частые вопросы

**Q: Я нажимаю «Новый», но ничего не происходит.**
A: Проверьте, что расширение загрузилось — в консоли (`~`) должно быть сообщение «CheckpointTimer v2.1 loaded». Если нет — проверьте путь к `lua/ge/extensions/checkpointTimer.lua` и наличие `scripts/checkpointTimer/modScript.lua`. Перезагрузите Lua: **Ctrl+L**.

**Q: Таймер запускается, но чекпоинты не фиксируются.**
A: Увеличьте **Радиус (м)** в настройках до 10–15. Либо проверьте, что подъезжаете к чекпоинтам **по порядку** (ЧК1 → ЧК2 → …).

**Q: Как отменить начавшийся заезд?**
A: Нажмите **«↺ Сброс»** в блоке «Заезд». Таймер обнулится, можно ехать заново.

**Q: Как удалить маршрут?**
A: В списке маршрутов кликните на красный крестик **✖** справа от названия.

**Q: Можно ли иметь несколько маршрутов на одной карте?**
A: Да, сколько угодно. Переключайтесь между ними кликом по названию в списке.

**Q: 3D-ворота не видны.**
A: Пошаговая проверка (v2.3):
1. Выберите маршрут в списке (клик по названию — должно подсветиться жёлтым). Ворота рисуются только для активного маршрута.
2. Убедитесь, что кнопка **«👁 Зоны: ВКЛ»** в настройках включена.
3. Нажмите **«🔧 Диагностика 3D»** — в логе (F11 → Console) появится отчёт + **над машиной появится пурпурная сфера на 5 секунд**.
4. **Если пурпурная сфера видна, а ворот нет** — значит debugDrawer работает, но координаты ворот далеко от камеры. Проверьте, что вы находитесь рядом с точкой старта активного маршрута (координаты старта логируются в диагностике).
5. **Если пурпурная сфера НЕ видна** — debugDrawer чем-то блокируется:
   - Проверьте в логе `getDrawingEnabled` — должно быть `true`
   - Попробуйте перезагрузить Lua: **Ctrl+L** + **F5**
   - Возможно, конфликт с другим модом, использующим debugDrawer
6. Проверьте, что в логе при загрузке мода было `debugDrawer: available (drawingEnabled=true)`.
7. Все методы `dd:drawLine`, `dd:drawCylinder`, `dd:drawSphere` должны быть `true` в диагностике.

**Q: После смены карты маршруты пропадают.**
A: Они не пропадают, а лежат в разных файлах — у каждой карты свой конфиг. Когда вернётесь на прежнюю карту, маршруты снова появятся.

**Q: Как экспортировать историю?**
A: Откройте файл `<userfolder>/settings/checkpointTimer/history.json` в любом редакторе. Формат — обычный JSON, можно конвертировать в CSV скриптом.

## 🔧 Технические детали

- **Lua VM**: GELUA (Game Engine Lua) — основной поток движка
- **UI**: AngularJS 1.5.8 (встроен в BeamNG)
- **Связь Lua → UI**: `guihooks.trigger('CTStatus', payload)` — push-модель, UI обновляется мгновенно
- **Связь UI → Lua**: `bngApi.engineLua('checkpointTimer.ct_command(...)')` — fire-and-forget
- **Время**: `be:getSimTime() / 1000` — секунды с учётом пауз симуляции (не реальное время)
- **Конфиги**: глобальные функции `jsonReadFile` / `jsonWriteFile` (BeamNG API)

## 📝 Что нового в v2.3 — ФИКС 3D-ворот!

**Корневая причина бага "ворота не видны":** debugDrawer в BeamNG требует **строгой типизации аргументов**. Хотя `pcall` возвращал `true` (нет Lua-ошибки), C++ binding рисовал пустоту, потому что я передавал `vec3` вместо `Point3F`, plain-строки вместо `String(...)`, и таблицы вместо `ColorF(...)`.

**Что исправлено:**
- ✅ Все позиции теперь конвертируются через `vec3(...):toPoint3F()` → `Point3F`
- ✅ Все цвета создаются через `ColorF(r,g,b,a)` и `ColorI(r,g,b,a)` (не через таблицы)
- ✅ Текст оборачивается в `String(...)` (не plain Lua string)
- ✅ При загрузке мода вызывается `debugDrawer:setDrawingEnabled(true)` — глобальный kill-switch
- ✅ После каждого draw вызывается `setLastZTest(false)` — маркеры не перекрываются геометрией (зданиями, рельефом)
- ✅ После каждого draw вызывается `setLastTTL(0.1)` — примитив живёт 100мс (анти-мигание при лагах)
- ✅ Диагностика `ct diag` теперь рисует **тестовую пурпурную сферу над машиной игрока** (живёт 5 сек) — сразу видно, работает ли debugDrawer в принципе
- ✅ Логируется `getDrawingEnabled()` + доступность `vec3`/`ColorF`/`ColorI`/`String`

**Как проверить, что фикс сработал:**
1. Обновите мод, перезагрузите Lua (**Ctrl+L**)
2. Выберите маршрут в списке (клик по названию)
3. В мире должны появиться ворота (зелёные/жёлтые/красные)
4. Если ворот всё ещё нет — нажмите **«🔧 Диагностика 3D»** в UI
5. В логе F11 ищите строку `Look for a PURPLE SPHERE above your car`
6. Если пурпурная сфера над машиной видна — debugDrawer работает, и ворота тоже должны быть видны (проверьте, что выбран маршрут)
7. Если сферы нет — пришлите лог диагностики, разберёмся

## ⚠️ Известные ограничения

- Триггер-зоны — **сферы**. Для более точного прохождения уменьшите радиус.
- Координаты — **мировые** (world space). Если в редакторе карт сдвинули геометрию — координаты в конфиге не обновятся автоматически, нужно перезаписать маршрут.
- История хранится в одном общем файле (`history.json`), не per-map. Фильтрация по карте — только в UI.
- Чекпоинты нужно проходить **строго по порядку**. Пропуск чекпоинта не засчитывается.
